## 🧭 介入判断フロー（Phase 7追加）

おやじストッパーの品質は「おやじ行動を検知できるか」より  
**「今ツッコんでいいかどうかを判断できるか」**にかかっている。

```
おやじ行動を検知
       ↓
偶発ダジャレ判定（意図的？偶発的？）
  ├── 偶発 → 「今の、ダジャレになってましたよ！」（カウントなし・即応答）
  └── 意図的 ↓
サイレントモード確認（手動オフ中？）→ YES → 介入しない
       ↓ NO
今日のモード確認（場の設定）
       ↓
直近の介入頻度確認（過去10分に2回以上 → 待機）
       ↓
おやじ行動の強度判定（軽い？重い？）
       ↓
ツッコミ実行 or 静観
```

### 各判断変数の詳細

| 変数 | 検知方法 | 備考 |
|---|---|---|
| サイレントモード | 手動フラグ | 1タップで即時ON/OFF |
| 場の種類 | Wi-Fi SSID + GPS | 自宅・職場は登録制、飲み屋はGPS自動判定 |
| 直近介入頻度 | ローカルログ | 短時間に重複しない |
| おやじ行動強度 | 頻度×種類スコア | 軽いものは閾値を上げる |
| 声のトーン | Pitch分析 | 深刻な会話中は静観 |

## 🌅 おやじデイリーリズム仕様（Phase 7追加）

### 朝：おやじ占い

**トリガー**：ユーザー設定時刻（デフォルト8:00）

**インプット**：
- 過去ログの曜日・時間帯別おやじ指数傾向
- カレンダー連携（飲み会予定日はリスク高）
- 直近3日間の平均おやじ度

**出力例**：
> 「今日のおやじ運：★★★☆☆　ダジャレが降ってきやすい一日です。昭和キーワードには要注意。ラッキーツッコミ：『はい3回目〜』」

**設計原則**：占い形式なので外れても笑える。プレッシャーゼロ。

---

### 夜：今日のおやじ総括

**トリガー**：ユーザー設定時刻（デフォルト22:00）

**出力例（通常日）**：
> 「今日のおやじ度：73ポイント。昨日より8ポイント上昇！ダジャレ4回、昭和ワード12回。でも誰も傷つけなかったね。明日も元気におやじってください。」

**出力例（ゼロポイントの日）**：
> 「今日はおやじ成分少なめでしたね。たまにはいいかも。ゆっくり休んでください。」

**出力例（落ち込みを検知した日）**：
> 「今日はちょっと大変な日だったみたいですね。おやじ度は関係なし。ゆっくり休んでください。」

**絶対ルール**：
- **沈黙は禁止**。どんな日も必ず声をかける
- 数値が低くても否定しない
- 「明日も頑張ろう」はノルマではなく励まし

## 🗺️ 場の検知仕様（Phase 7追加）

### 検知方法

| 場の種類 | 主な検知方法 | 精度 |
|---|---|---|
| 自宅 | 自宅Wi-Fi SSID登録 | ★★★ |
| 職場 | 職場Wi-Fi SSID登録 | ★★★ |
| 飲み屋 | GPS（飲食店）+ 夜間時間帯 | ★★☆ |
| 移動中 | GPS移動速度 | ★★★ |
| その他 | 未登録Wi-Fi + 非移動 | ★☆☆ |

### 場ごとのデフォルト設定

| 場 | デフォルトアバター | ツッコミ強度 | 備考 |
|---|---|---|---|
| 自宅 | 孫キャラ／おかん | 中 | 家族想定 |
| 職場 | 若者後輩 | 弱〜中 | パワハラリスク配慮 |
| 飲み屋 | 飲み屋の友達／昭和司会者 | 強 | 最もツッコみやすい場 |
| 移動中 | なし | オフ | 公共の場 |

### モード切替の設計

ユーザーは以下から選択可能：

- **自動モード**：場の検知に応じて自動切替
- **手動モード**：常にユーザーが手動で切替
- **自動＋手動上書き**（推奨）：普段は自動、必要な時だけ手動変更

## 📊 メトリクス計算式

```
昭和度     = (昭和キーワード出現数 / 総単語数) × 100
おやじ指数 = (ギャグ回数×3) + (繰り返し×2) + 絵文字数
ギャグ連発 = 過去180分間のダジャレ検出数
```

**安全圏**：昭和度 < 70%、おやじ指数 < 100、ギャグ < 5回/3時間
